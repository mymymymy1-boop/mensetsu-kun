## 重要な設計前提（合理判断・リサーチで検証する）

1. **音声出題(TTS)**: Web Speech API `SpeechSynthesis`。日本語音声・ローカル合成中心で privacy リスク低。→ [要V2確認: 各ブラウザの日本語音声品質]
2. **音声回答(STT)**: Web Speech API `SpeechRecognition`。Chrome/Edgeで日本語対応だが**音声がブラウザのクラウド認識に送られる場合がある**。子供の声を扱うため、(a) 既定で録音を保存しない (b) プライバシー注意を保護者に明示 (c) STTは「補助」と位置づけ、採点は録音の有無に依存しない設計にする。→ [要V2確認: SpeechRecognitionの送信先・対応ブラウザ]
3. **採点**: 5〜6歳の発話はSTT精度が不安定。採点は「自動(認識キーワード/応答長/沈黙時間)＋保護者による手動補正」のハイブリッド。**点数で子供を否定しない**＝ポジティブ・フィードバック基調。
4. **質問レベル**: 4校の難度・傾向に合わせて出し分け（慶応系=自己/家庭/思考を問う深い質問が多い、行動観察系=協調/マナー/とっさの受け答え）。リサーチで確定。
