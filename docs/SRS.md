## 1. はじめに
### 1.1 目的
私立小学校受験を控える年長児（5〜6歳）が、家庭で「面接・行動観察での受け答え」を音声で繰り返し練習できるWebアプリを定義する。音声で質問を出題し、子供が声で答え、その回答を聞き取り（任意）・採点し、保護者がフィードバックを得る。

### 1.2 範囲
4校（慶応横浜初等部・慶応幼稚舎・桐蔭学園小・森村学園初等部）＋共通＋行動観察声かけの質問集を収録。スコープ詳細は `scope_table.md` を正本とする。

### 1.3 用語集
| 用語 | 定義 |
|---|---|
| TTS | Web Speech API SpeechSynthesis による音声読み上げ（出題） |
| STT | Web Speech API SpeechRecognition による音声認識（任意の聞き取り） |
| 録音 | MediaRecorder API によるローカル録音（外部送信なし） |
| 保護者ゲート | 子供が誤って到達できないよう大人向け操作で保護した領域 |
| お尋ね | 絵画・制作中に試験官が子供に作品について問いかけること（慶応2校の実態） |
| 信頼度 | 情報源の確からしさ A(公式)/B(大手教室・技術一次)/C(体験談)/D(出所不明) |

### 1.4 参考文書
`research/research_v1.md` `research/research_v2.md` `research/question_bank.md` `intake_sheet.md` `scope_table.md`

## 2. システム概要
- **形態**: React + Vite + TypeScript の静的SPA。サーバー・DB・外部API・課金・アカウントなし。ブラウザのみで完結。
- **主要機能**: ①校・モード選択 ②音声出題(TTS) ③回答取得(録音＋任意STT) ④採点(ヒューリスティック＋保護者補正) ⑤子供向けポジティブFB ⑥保護者向け結果 ⑦保護者ゲート ⑧質問データ(出典・信頼度・レベル付き)。
- **コンテキスト図（テキスト）**:
```
[子供] --音声/タップ--> [ブラウザSPA] --TTS--> [ブラウザ音声合成(ローカル)]
                              |--録音--> [MediaRecorder(ローカルのみ)]
                              |--任意STT(同意時)--> [ブラウザ音声認識サービス(Google等/外部)]※既定オフ
[保護者] --ゲート通過--> [結果/設定/採点補正]
[質問データ(静的JSON・出典/信頼度付)] --> [SPA]
```

## 3. ステークホルダー
| 主体 | 関心事 |
|---|---|
| 子供(5〜6歳) | 一人で操作できる・楽しい・否定されない |
| 保護者 | 正確な受験情報・練習の振り返り・子供の声を外部に出さない安全性・無料 |
| 学校(間接) | 受験情報の正確な引用・出典明示（誤情報拡散の防止） |
| ブラウザ提供元(外部) | Web Speech/MediaRecorder 標準APIの提供元（規約遵守） |

## 4. 運用サイクル
- 日次: 保護者が校・モードを選び、子供が5〜10分（複数質問）練習→結果確認。
- 随時: 保護者が質問データの信頼度・出典を確認、設定（STT有効/無効・音声速度・保存可否）を調整。
- 定期: アプリ内「最新は公式要項で確認」案内に従い、保護者が各校公式を確認（年度変動対応）。

## 6. 外部インターフェース
### 6.1 外部API: なし（ブラウザ標準APIのみ）
| API | 用途 | 認証 | 料金 | プライバシー |
|---|---|---|---|---|
| SpeechSynthesis | 出題読み上げ | なし | 無料 | 子供の声を扱わない（低リスク） |
| MediaRecorder/getUserMedia | 回答録音 | マイク許可 | 無料 | ローカルのみ・外部送信なし |
| SpeechRecognition(任意) | 回答の自動聞き取り | マイク許可 | 無料 | **既定で音声を外部送信。保護者同意ゲートの奥でのみ有効** |
| localStorage(任意) | 結果保存 | なし | 無料 | 端末内のみ・音声は保存しない |

### 6.2 UI
子供向け（大ボタン2cm四方・単一タップ・ひらがな・音声主導・キャラ）と保護者向け（ゲートの奥・結果/設定）を分離。詳細は第8章。

## 7. 機能要件（FR）

### FR-SEL-001: 学校を選択する
- 優先度: **Must**
- Given: ホーム画面が表示されている
- When: 保護者または子供が学校カード（慶応横浜初等部/慶応幼稚舎/桐蔭学園小/森村学園初等部/共通/行動観察）の1つをタップする
- Then: 選択した学校に紐づく質問セットが読み込まれ、モード選択またはセッション開始へ遷移する／各カードはタップで音声読み上げされる
- 例外: 質問データ読込失敗時は「もういちど」案内を表示し、他校選択は維持

### FR-SEL-002: 練習モードを選択する
- 優先度: **Must**
- Given: 学校が選択されている
- When: 「めんせつ（面接）」または「こうどうかんさつ（声かけ）」モードをタップする
- Then: 該当モードの質問のみが出題対象になる（慶応2校では面接モードは「制作お尋ね」中心と表示）
- 例外: 選択校に該当モードの質問が0件なら、その旨を表示し選択不可にする

### FR-AUD-001: 質問を音声で読み上げる
- 優先度: **Must**
- Given: セッションが開始され質問が画面に表示されている
- When: 出題が提示される、または「もういちど きく」をタップする
- Then: SpeechSynthesisでja-JP音声により質問文を読み上げる（rateは既定0.9）／読み上げ中はその旨を視覚表示
- 例外: ja-JP音声が無い/TTS非対応の場合、文字（ひらがな大）表示＋効果音でフォールバックし、処理は継続

### FR-REC-001: 子供の回答を録音する
- 優先度: **Must**
- Given: 質問が読み上げられ、マイク利用が許可されている
- When: 子供が大きな「マイクボタン」をタップして話し、再度タップ（または自動無音検出）で終了する
- Then: MediaRecorderで回答音声をローカル(メモリ/Blob)に保持し、保護者画面で再生できる／音声は外部送信しない
- 例外: マイク不許可/非対応時は録音をスキップし「読み上げ＋口頭練習（保護者が採点）」モードで継続

### FR-STT-001: 子供の回答を自動で聞き取る（任意・同意制）
- 優先度: **Should**
- Given: 保護者ゲートを通過し、保護者がSTTを明示的に有効化（同意文に同意）している、かつブラウザがSTT対応
- When: 子供が回答を話す
- Then: SpeechRecognition(ja-JP)で認識テキストを取得し採点の補助入力にする／有効化前に「お子さまの声がブラウザの音声認識サービス（Google等）に送信されます」と明示する
- 例外: 非対応/エラー(no-speech/network/not-allowed)時は自動で `record-only` にフォールバックし、採点は保護者補正に委ねる

### FR-SCORE-001: 回答を採点する（ヒューリスティック）
- 優先度: **Must**
- Given: 1問への回答が完了している（録音有無・STTテキスト・応答時間が得られている）
- When: 回答が確定する
- Then: 内部スコア(0〜100)と観点別結果（聞こえたか/答えられたか/応答長/想定キーワード一致/沈黙時間）を算出する／STT無効時はキーワード一致を「未判定」とし保護者補正前提にする
- 例外: 入力が空（無回答）でもエラーにせず、スコア0・「もういちど やってみよう」を返す

### FR-SCORE-002: 保護者が採点を手動補正する
- 優先度: **Must**
- Given: 保護者ゲートを通過し結果画面を見ている
- When: 保護者が各問のスコア/評価を上書きし、メモ（姿勢・声の大きさ等）を入力する
- Then: 自動採点結果を保護者の補正値で上書きし、セッション結果に反映する（自動採点はあくまで補助）
- 例外: 入力がなければ自動採点値を維持

### FR-FB-001: 子供にポジティブなフィードバックを表示する
- 優先度: **Must**
- Given: 1問の採点が完了している
- When: 結果が子供向けに提示される
- Then: **数値の点数を子供に見せず**、花丸/star/キャラのほめ言葉（音声＋アニメ）で表現する／未回答・低スコアでも否定せず「おしい！ もういちど やってみよう」と前向きに誘導する
- 例外: なし（常に肯定的表現）

### FR-RPT-001: 保護者にセッション結果を表示する
- 優先度: **Must**
- Given: セッション（複数問）が終了し保護者ゲートを通過している
- When: 保護者が結果画面を開く
- Then: 問題ごとのスコア・観点・録音再生・出典/信頼度・保護者メモを一覧表示し、セッション合計とアドバイス（観点別）を表示する
- 例外: 結果が0問なら「まだ れんしゅうが ありません」を表示

### FR-GATE-001: 保護者領域をゲートで保護する
- 優先度: **Must**
- Given: 設定・結果詳細・STT有効化など大人向け機能にアクセスしようとしている
- When: 子供/利用者が保護者領域への入口をタップする
- Then: 簡単な計算問題（または長押し2秒＋数字入力）を提示し、正解時のみ保護者領域へ入れる
- 例外: 不正解は再試行を促し、保護者領域へは入れない

### FR-DATA-001: 出典・信頼度・レベル付きで質問を提供する
- 優先度: **Must**
- Given: アプリが起動している
- When: 質問データを読み込む
- Then: 各質問が `id/school/category/level/text(ひらがな)/intent/source(URL)/reliability(A-D)/followUps/sampleGoodAnswer` を持ち、保護者画面で出典と信頼度を表示できる
- 例外: 不正データはスキーマ検証で除外し、その旨をログに記録

### FR-DATA-002: 校ごとの傾向に合わせて出題する
- 優先度: **Should**
- Given: 校が選択されている
- When: 出題順を決める
- Then: 慶応2校は「制作お尋ね＋共通」中心、桐蔭は「思考・とっさhard」を厚く、森村は「自己紹介easy＋制作お尋ね」中心、共通はeasy〜mid中心となるよう、校別の重み/フィルタで選択する
- 例外: 該当質問が不足する場合は共通カテゴリで補完

### FR-SYS-001: 最新情報の確認を促す注意を常時表示する
- 優先度: **Must**
- Given: 校選択・結果画面を表示している
- When: 受験情報を提示する
- Then: 「面接の有無・形式は年度で変わります。最新は各校公式の募集要項でご確認ください」を保護者向けに常時表示する
- 例外: なし

### FR-SYS-002: 音声を既定で保存しない
- 優先度: **Must**
- Given: 録音・STTが行われている
- When: セッションが終了する、またはページを離れる
- Then: 録音音声・STT原文は既定で永続保存せず破棄する／保存はlocalStorageのスコア/メモのみで、保護者が明示的に有効化した時だけ
- 例外: 保護者が「結果を保存」を明示ONにした場合のみスコア/メモを保存（音声は保存しない）

## 8. 画面・操作フロー
```
[ホーム]→[校選択]→[モード選択]→[出題画面(問1..N)]→[セッション結果(子供向けFB)]
   └(右上隅)→[保護者ゲート]→[保護者画面: 結果詳細/設定/STT同意/出典確認]
出題画面: 中央キャラ + 質問(音声+ひらがな) + 大マイクボタン + 「もういちど きく」 + 録音中アニメ
```
- 子供向け: 単一タップのみ・ドラッグ/スワイプ/長押し不使用・1画面1アクション・無反応時は質問自動再読。
- 保護者ゲート: ホーム右上隅の小さな入口。

## 9. データ要件
| データ | 形式 | 保存 | ライフサイクル |
|---|---|---|---|
| 質問データ | 静的JSON(src/data) | アプリ同梱 | ビルド時固定・出典/信頼度付 |
| 回答音声 | Blob(メモリ) | **保存しない** | セッション中のみ・終了で破棄 |
| STT認識テキスト | 文字列(メモリ) | **保存しない** | 採点後破棄 |
| セッション結果(スコア/観点/メモ) | JSON(localStorage) | **任意・既定オフ** | 保護者が有効化時のみ |
| 設定(STT有効/速度/保存可否) | JSON(localStorage) | 端末内 | 変更まで保持 |

## 11. セキュリティ・監査
- SEC-001: 子供の音声はローカル処理を既定とし、外部送信はSTT同意時のみ。送信先をUIで明示。
- SEC-002: 音声・STT原文を永続保存しない。
- SEC-003: 保護者ゲートで設定/結果/STT有効化を保護（子供の誤操作・誤同意防止）。
- SEC-004: 外部アカウント・サーバー・課金なし＝攻撃面が最小。
- 監査: クライアントのみのためサーバー監査ログは無し（N/A）。設定変更は端末内のみ。

---

## 12. 制約条件
- 技術: 外部APIキー無しで動作（ブラウザ標準API）。STTは実質Chrome/Safariのみ・5〜6歳のWERが高い（自動採点を主軸にしない）。
- 法的/倫理: 受験情報は出典明示・引用範囲で使用。学校公式が逐語非公開のため、質問はB/C由来であることを明示。子供向けは点数で否定しない。
- 運用: 年度で面接形式が変わるため公式確認を促す。
- 予算: ¥0（無料・キー不要）。

## 14. リスク・変更管理
リスク台帳の正本は `research/research_v2.md §5`。主要:
| RISK | 影響 | 対策 | 対応FR |
|---|---|---|---|
| RISK-001 子供の声の外部送信 | 高 | 既定record-only・STT同意制・送信先明示 | FR-STT-001/FR-SYS-002/NFR-SEC-001 |
