## 2. 技術スタック（確定）

| レイヤー | 技術 | バージョン | 選定理由 |

| UI | React | 18.x | 子供向けコンポーネント・状態管理が容易・エコシステム充実 |
| ビルド | Vite | 5.x | 高速・静的サイト出力・キー不要でローカル/GitHub Pages配信可 |
| スタイル | CSS（プレーン or CSS Modules） | - | 依存最小。大ボタン・ひらがな・配色を直接制御 |
| TTS | Web Speech API SpeechSynthesis | ブラウザ標準 | キー不要・無料・ja-JP・出題読み上げ |
| 録音 | MediaRecorder API | ブラウザ標準 | キー不要・ローカル録音・外部送信ゼロ（保護者採点の基盤） |
| STT(任意) | Web Speech API SpeechRecognition（アダプタ化） | ブラウザ標準 | 「聞き取り」をしたい場合の任意機能。**保護者同意ゲートの奥でのみ有効化**。将来Vosk-browser(ローカル)へ差し替え可能なアダプタ設計 |
| 採点 | 自作ヒューリスティック + 保護者補正 | - | キーワード/回答有無/応答長/沈黙。LLM意味採点はDEFER（キー導入時にアダプタ差し替え） |

| テスト | Vitest + Testing Library + Playwright(E2E) | 最新 | unit/integration/e2e。音声I/Oはモック化して決定論テスト |

## 3. アーキテクチャ判断（音声I/O・採点）

### 3-1. 音声入力モード（feature flagで切替・既定= 安全側）
| モード | 内容 | プライバシー | 既定 |
|---|---|---|---|
| `record-only`（既定推奨） | MediaRecorderで録音→保護者が再生して採点 | ◎ 外部送信ゼロ | ★MVP既定 |
| `webspeech-stt`（任意） | 録音＋Web Speech STTで自動聞き取り→ヒューリスティック採点。**有効化前に保護者へ「お子さまの声がブラウザの音声認識サービス(Google等)に送信されます」と明示し同意必須** | △ 外部送信あり | 保護者が明示ONした時のみ |
| `local-stt`（DEFER） | Vosk-browser等でローカル認識 | ◎ | 将来 |

→ **STTは `SttAdapter` インターフェースで抽象化**。実装を `RecordOnlyAdapter` / `WebSpeechSttAdapter` / (将来)`VoskAdapter` に差し替え可能にする。これによりキー無し・プライバシー優先で全テストが通る設計（DRY_RUN相当のモックアダプタも用意）。

### 3-2. 採点ロジック（教育的配慮を内蔵）
- 入力: 想定回答パターン（質問データに付与するキーワード/最低語数/カテゴリ）＋ 実際の回答（録音有無・STTテキスト・応答時間）。
- 出力(保護者向け): スコア(0〜100の内部値) + 観点別チェック（聞こえた/答えられた/長さ/キーワード一致/姿勢メモ欄）。
- 出力(子供向け): **点数を見せず**、花丸・star・キャラのほめ言葉のみ。未回答/誤りも否定せず「もういちど やってみよう」。
- 保護者が手動でスコアを上書き・メモ追加可能（自動採点はあくまで補助）。

### 3-3. 質問データ設計（出典・信頼度・レベルを内蔵）
各質問オブジェクトに: `id / school(4校 or common / behavior) / category / level(easy|mid|hard) / text(ひらがな) / intent(評価観点) / sampleGoodAnswer / followUps / source(URL) / reliability(A|B|C|D)` を持たせる。**画面とデータに信頼度を明示**し、誤情報リスクを抑える（黄金ルール: 根拠IDの精神をアプリ層にも適用）。

## 6. SRSへの入力事項（Phase 2で使う確定情報）

1. **対象**: 年長(5〜6歳)の子供本人＋保護者。形式=クライアントサイド静的Webアプリ(React+Vite+TS)。
2. **必須機能**: 校選択(4校+共通+行動観察)/モード選択/音声出題(TTS)/回答取得(録音、任意でSTT)/採点(ヒューリスティック+保護者補正)/子供向けポジティブFB/保護者向け結果画面/保護者ゲート。
3. **質問データ**: 4校別＋共通＋行動観察声かけ。各質問に出典・信頼度・レベル・評価観点。慶応2校は「面接なし・制作お尋ね」、桐蔭は「親子面接・口頭試問・とっさ判断」、森村は「グループ面接・制作中質問」をレベル/傾向に反映。
4. **非機能**: 子供が一人で操作可(2cm四方ボタン・単一タップ・音声主導・ひらがな)/音声を外部保存しない/対応ブラウザ案内とフォールバック/短セッション。
5. **安全/倫理**: 子供の声の外部送信は同意制・既定オフ。点数で子供を否定しない。受験情報は出典・信頼度明示＋公式確認の注意喚起。
6. **テスト**: 音声I/Oはモックアダプタで決定論テスト。採点ロジック・質問選択はunit。1サイクルはe2e。

### 校別の出題傾向まとめ（質問データのレベル設計指針）
- **慶応幼稚舎/横浜初等部**: 面接なし。「制作お尋ね」型（何を作った？どうしてこの色？どんなお話？）＝表現力・対話。**common+作品説明カテゴリ**を中心に、行動観察の協調性を補助。
- **桐蔭学園小**: 親子面接＋個別口頭試問。自己紹介・がんばっていること・とっさの判断(雲が何に見える/動画でこぼした時どう声かけ)＝**思考・とっさカテゴリのhardを厚く**。
- **森村学園初等部**: グループ面接＋制作中質問(描いたもの/誰とどこで食べたい)＝**自己紹介easy＋制作お尋ね＋好き理由**。
- **共通(全私立)**: 自己紹介/家庭/園生活/友達社会性/生活習慣/将来。easy〜mid中心。
- **行動観察声かけ**: 「今何してるの/楽しい？/どうしたらいい？」＝信頼度明示しつつ、保護者解説で観点(協調性等)を伝える。
